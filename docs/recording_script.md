# 录屏讲稿：基于规则推理的狼人杀局势分析助手

## 0:00–0:30 项目定位
我做的是一个基于规则推理的狼人杀局势分析助手。它对应课程中的知识表示、知识库、事实、规则和前向链推理。系统不直接判断谁一定是狼人，而是识别身份冲突、查验矛盾、投票关注点，并展示推理过程。

## 0:30–1:15 设计说明
系统把玩家声明、查验结果、投票记录和死亡信息表示为四类 facts。规则库分成 hard logic、dependent claim 和 soft attention 三层。推理引擎会反复扫描规则，满足条件时生成新的中间事实，并记录 trace。

## 1:15–2:10 默认案例演示
打开 index.html，点击加载默认案例，再点击运行推理。默认案例中 P1 和 P2 都声明自己是预言家，因此触发 R1 形成对跳；他们对 P3 给出相反查验，所以触发查验矛盾；身份冲突和查验冲突结合后，系统识别 P1/P2 是核心对跳冲突。投票方面，P4 投 P1、P5 投 P2，说明投票围绕对跳双方分裂。

## 2:10–3:00 手动输入案例
接下来清空事实，手动输入一个没有内置在代码中的例子。仍然让 P1/P2 对跳，但让 P4 和 P5 都投给 P1，P6 投给 P2。运行推理后，系统不仅识别核心对跳，还会识别 P1 被多人投票，并作为核心冲突玩家处于投票压力下。这说明系统不是针对默认案例写死，而是根据用户输入动态推理。
